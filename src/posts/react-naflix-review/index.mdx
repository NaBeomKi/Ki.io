## Netfilx clone...?

사실 넷플릭스 클론이라기에는 차이가 꽤 많아서 넷플릭스 클론이라고 불러도 되는 건지는 잘 모르겠네요.
기능 자체는 현재 서비스 되고있는 왓챠피디아와 더 유사한 느낌이긴 합니다.(그런데 이제 리뷰 남기기와 별점 매기기들의 회원 활동이 빠진...)

자세히 설명하자면 영화 및 TV 프로그램 등의 간략한 정보들을 조회할 수 있는 사이트입니다.
그리고 그런 영상매체를 다루는 사이드 프로젝트들의 대명사처럼 되어버린 넷플릭스...

## 기획

몇개월전 React를 배우기 시작하며 노마드코더 사이트의 강의를 수강했었는데, 이번 프로젝트는 해당 강의를 기반으로 하고 있습니다.
강의에서는 Class 기반의 컴포넌트와 디자인 패턴으로 프로젝트를 진행했던 부분들을 Function 기반의 컴포넌트와 Hooks로 리팩토링했고,
추가로 useReducer와 useContext를 따로 익혔기 때문에 이번에 적용해봤습니다.
그리고 강의에서 적용한 CSS도 왓챠를 참고해 약간 수정했습니다.

영상정보는 [TMDB](https://www.themoviedb.org/)의 api를 이용했습니다.

## 느낀점

지난 프로젝트를 리팩토링 하는 프로젝트라고 밝히긴 했습니다만,
몇개월(1년이 다되어가는듯 합니다)전에 수강한 강의를 혼자서 바닥부터 다시 만든 프로젝트였기에 거의 한달 가량 걸린 듯 합니다.
게으름 피운걸 감안하더라도 빠르게 끝낼 수 없었을 것 같아요.

저 자신도 리팩토링 위주라고 생각하고 시작한 프로젝트지만,
이번 프로젝트에서 가장 큰 성취는 새로 익힌 useReducer, useContext를 적용한 것보다도
생각지도 못했던 Route 부분인 것 같아요.

React Router Dom의 BrowserRouter, Switch, Route를 사용하면서 다시 사용방법을 정리할 수 있었습니다.
특히 어려움을 겪었던 부분은 쿼리 스트링 별로 다른 컴포넌트를 렌더링 하는 부분이었는데,
이 부분을 처리하기 위해 BrowserRouter, Switch, Route을 사용하려고 했지만 그렇게 되지 않는다는 걸 알았습니다...
withRouter를 이용해 location.search에서 내용을 가져와 if문으로 처리해야한다는 걸 깨달았습니다.

이번 React 프로젝트를 하고나서 지난 React를 이용한 프로젝트들을 리팩토링 해야겠다는 생각이 들었습니다.
이전에 했던 covid19 사이트와 블로그까지 모두 리팩토링을 해야할 것 같아요.
조만간 업데이트 된 블로그를 보여드릴 수 있으면 좋겠네요.

마지막으로 [Nafilx](https://ki-react-naflix.netlify.app/)도 구경해주세요~
